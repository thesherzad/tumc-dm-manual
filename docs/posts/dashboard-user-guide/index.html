Introduction

The purpose for the dashboard is to provide quick and near real time access to summaries and reports from the data in the database. Additionally, some tools to automate some tasks.

Getting Started

  • Accessibility: the dashboard is web based app, at the time of writing this manual, it’s hosted on shinyapp.io, accessible via this link.

  • Log-in: to use the dashboard, you need to have a pre-defined username and password. Admin can help with username and password. There’s no user based roles, that means any user who logs in, they’ll have access to the same features. So, only one username and password can be used by all authorized users.

  • Log-out: at the right-bottom side of the screen, there’s a floating button, hover your mouse on it and you’ll see a log-out option.

Important

It’s important to note that this app is hosted on a free plan, so there’s usage limitation. When you don’t need to use the dashboard, you may close the window.

Interface Overview

Features is divided into multiple tabs for ease of finding and better user experience.

Home

This tab includes summaries and graphs representing from multiple datasets, including Households/Guests, Appointments, Distributions, ESOL and such.

Appointments

  • Appointment Filters: you may use the available filters to subset or see specific appointments, or appointments for certain date (or a date ranges), and so on.

    Tip

    If you set a filter, it’ll affect all the visible graphs and tables under Appointment tabs.

    If you want to download appointments for a certain service and a date range, you can apply both at the same time. E.g. Choose the desired date range, and then select the desired service. The resulting graph and table will be based on that filter.

  • Appointment Calendar: it can be helpful to show you how many appointments you should expect for certain day, and time slot.

  • Appointment Details: this table view is helpful if you want to download or view your desired appointments in more details, e.g. date, time slots and such.

Distributions

Use the filter options to select your desired subset of distributions, e.g. for certain month or quarter or so. Or subset to see the details for a certain service, such as food or diaper.

The numbers are graphs will change based on the filters instantly.

Households

The term “Households” and “Guests” will be used interchangeably throughout this manual. Both of them point to the same table in the database. Under this tab, you’ll see filter options that you can use to filter to your desired subset of the data.

Volunteers

As of now, we don’t have any data or practical use of it, but this space is to show summaries about volunteers availability.

The email section should work. It can be used to send emails to a group of volunteers. The purpose for this feature is automatically notify volunteers about their upcoming shift - but we don’t have such data yet.

Note

It uses a non-official email address to send emails. So, when volunteers receive email, it’d be from a G-mail account. If you want to use this to send emails, make sure you include enough introduction.

Tools

  • Download Filter Options: the filter options can be used to select a desired dataset and subset it by column and/or a date range. Note that, this filter option is only applicable on the subsequent data table.

  • Dataset Previous: it will show the resulting dataset from the defined filters.

  • Upload Distribution Data to KOBO: this feature is to help with upload distribution data to kobo server. This feature is specifically designed for Tuesday distributions.

    • Input: a csv file. Download the distribution google sheet as csv (the check boxes should convert to TRUE and FALSE)

    • Distribution Date: choose the date of distribution

    • When ready, click on the Upload button to submit the distributions record to kobo

    Note

    Please note that the Upload button will disable after upload action is successful. That’s because to avoid accidental resubmit. If you really have multiple files to upload, you’ll need to refresh your page and restart the upload.

Triny

Triny is the nickname of the AI tool embedded in the dashboard. This tool is helpful to generate descriptive summaries from the available datasets that are not in the dashboard. It works the same way as public chatbot (e.g. ChatGPT) but this is aware of the available datasets and can summarize the datasets to answer your questions.

If you get to use this tool, you might find it way faster to generate summaries and get answers quickly to your questions.

Example queries

  • How many appointments do we have tomorrow, and how many of them would need diaper and what sizes of diapers?

  • How many distributions we’ve had in the past week and how many kids benefited from these distributions.

Note

This is a paid service, if you’ve general question that are not related to datasets, it’s not recommended to use this tool.

Warning

While the AI doesn’t have access to the database or data instead it generates and runs code to answer your query, but like any other public AI tool, this tool is not safe to share private or personal data, such as address, phone number, credit card and such.
